cli.rs文件

文件职责：启动时打印一个漂亮的终端 Banner
进入命令行对话循环，也就是 REPL
把用户输入发送给本地 Ollama 模型，并把回复打印出来

这些 const 是终端样式
const APP_NAME: &str = "A U R O R A";
const APP_TAGLINE: &str = "local-first assistant shell";
const RESET: &str = "\x1b[0m";
const DIM: &str = "\x1b[2m";
const BOLD: &str = "\x1b[1m";
const ACCENT: &str = "\x1b[38;5;215m";
const MUTED: &str = "\x1b[38;5;246m";

这些不是业务逻辑，主要是 ANSI 终端颜色控制码。

\x1b[1m

代表加粗。

\x1b[0m

代表重置样式。

所以它想做的效果是：终端里显示一个有颜色、有标题、有分隔线的 CLI 界面。

run是这个CLI函数的入口

第一步render_banner(...)打印启动页面，第二步repl_loop(config)，启动真正的对话循环

获取终端宽度
let width = env::var("COLUMNS")
    .ok()
    .and_then(|value| value.parse::<usize>().ok())
    .unwrap_or(80)
    .min(88);

流程是：

读取环境变量 COLUMNS
如果读取成功，就转成 usize
如果失败，就默认 80
最大不超过 88

也就是说：

终端宽度 = COLUMNS 的值
如果没有，就用 80
如果太宽，最多按 88 算

repl_loop：核心对话循环

REPL 的意思是：

Read → Eval → Print → Loop
读取输入 → 执行处理 → 打印结果 → 继续循环

读取用户输入
let mut input = String::new();
stdin
    .read_line(&mut input)
    .map_err(|err| format!("failed to read input from terminal: {err}"))?;

这里创建一个空字符串：
    let mut input = String::new();
    然后把用户输入的一整行读进去。

let trimmed = input.trim();

把前后的空格、换行去掉。
    
清空会话
if trimmed == "/clear" {
            session.clear();
            println!("助手> 已清空当前会话。");
            continue;
        }

调用 Ollama 模型
最核心的是这里：

println!("助手> 正在思考...");
match ollama::chat(&config.ollama_url, &config.model, session.history(), trimmed) {

这里调用：传进去四个东西：

&config.ollama_url

Ollama 服务地址，比如可能是：

http://localhost:11434
&config.model

模型名，比如：

llama3
qwen
mistral
session.history()

当前对话历史。

trimmed

用户这一次输入的内容。

所以这行的意思是：

把当前模型地址、模型名、聊天历史、用户新问题一起发给 Ollama

这个文件理解成这样：

run(config)
  ↓
打印 Banner
  ↓
进入 REPL 循环
  ↓
显示 >
  ↓
读取用户输入
  ↓
判断特殊命令
    - 空输入：跳过
    - quit / exit：退出
    - /clear：清空会话
  ↓
调用 ollama::chat
  ↓
拿到模型回复
  ↓
保存到 Session
  ↓
打印回复
  ↓
回到下一轮输入
