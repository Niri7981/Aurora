# AuroraPulse 路线图

## 产品方向

AuroraPulse 是面向 AI Agent 的本地个人记忆层。

> 把自己告诉 Aurora 一次，所有你授权的 AI 都能认识你。

项目不再建设独立聊天 Agent、模型提供方、桌面工具执行器或语音助手。当前和未来工作只围绕三件事展开：本地个人记忆、受控上下文披露、跨 Agent 使用。

## 当前状态

截至 2026-07-20，MCP Context Gateway 已完成：

- 本地身份、当前关注、偏好和隐私文件
- `get_identity`
- `get_current_focus`
- `search_personal_context`
- 任务级 `ContextPack`
- 动态隐私标记过滤
- 稳定来源 URI 与省略信息
- 本地 JSONL 访问审计
- Codex stdio MCP 真实接入验证
- `init`、`preview`、`audit` 管理命令

## Phase 5：持久个人记忆

目标：让 Aurora 不再只依赖几份手工文件，而是拥有可纠正、可追溯、可过期的结构化记忆。

首版记忆字段：

- 类型：身份、偏好、经历、关系、当前目标、开放事项
- 内容与主体
- 来源引用
- 创建时间与有效时间
- 可信度
- 敏感等级
- 状态：候选、已确认、已拒绝、已过期
- 修订和删除记录

验收标准：

- 用户可以新增、检查、纠正、过期和删除记忆
- 每条记忆都能回答“来自哪里”
- MCP 检索只返回当前有效且符合披露策略的记忆
- 删除后不会继续从索引或缓存中返回

## Phase 6：通用导入与聊天记录

目标：建立 Source Adapter，而不是把每种聊天格式直接写进核心逻辑。

实现顺序：

1. 定义标准 `SourceRecord` 和 `NormalizedMessage`
2. 支持通用 JSON/TXT 聊天记录
3. 保存原始来源哈希、会话、说话人和时间
4. 从来源生成待审阅记忆候选
5. 增加微信聊天记录 Adapter
6. 再扩展 GPT、Gemini、邮件、笔记和文档来源

原始聊天默认不会直接通过 MCP 披露，也不会自动成为用户身份。

## Phase 7：真实授权系统

目标：把“在 Host 中注册 MCP”升级为 Aurora 自己可管理的授权边界。

- 客户端身份
- 工具和数据范围
- 按来源与敏感等级授权
- 临时授权与永久授权
- 撤销与重新授权
- 拒绝事件审计

## Phase 8：更强检索

只有在真实个人记忆数量需要时再加入：

- 全文索引
- 语义检索
- 时间和有效性过滤
- 冲突记忆检测
- 来源质量与置信度排序

向量数据库不是产品起点。可纠正的记忆模型和可靠来源优先。

## 当前下一步

先完成 Phase 5 的最小闭环：

```text
手动新增一条记忆
    -> 保存来源和有效期
    -> 用户检查并纠正
    -> 新 Codex 任务通过 MCP 找到它
    -> 审计记录此次披露
    -> 用户删除后再次检索不到
```
